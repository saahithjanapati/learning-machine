# Scale AI Research Internship Prep Index

This is a cross-filing index for papers and resources that may be useful for a Scale AI research internship, especially around post-training, interpretability, and their overlap.

Primary filing stays wherever the item normally belongs. This index adds an additional research-prep layer so one paper can be both a regular AI lesson and part of a focused internship reading queue.

## Category Rules

- `post-training`: SFT, RL, RLVR, preference optimization, reward modeling, verifier design, rubric rewards, agent self-improvement, and post-training infrastructure.
- `interpretability`: feature readouts, probes, activations, representation geometry, model diffing, internal-state diagnostics, and mechanistic or behavioral transparency.
- `embeddings`: embedding models, representation geometry, similarity metrics, retrieval/evaluation spaces, semantic distance, style distance, and other vector-space signals.
- `agent-memory`: context engines, trace-to-memory pipelines, retrieval-augmented agent memory, production feedback loops, and enterprise-specific procedural memory.
- `evaluation`: benchmark design, LLM-as-judge, rubric scoring, workflow harnesses, regression checks, and counterfactual task replay.
- `post-training x interpretability`: methods that use features, probes, rubrics, latent concepts, or interpretability-style diagnostics as supervision, rewards, monitors, or alignment-generalization tools.
- `scale-context`: Scale Labs papers, Scale-adjacent topics, or papers that closely match Scale research themes such as evaluation, data, verifiers, agents, and scalable supervision.

## Open Questions To Track

- When can a learned reward model replace exact verifiers, and when does it only hide reward hacking?
- Which open-ended behaviors are already represented well enough internally to support feature rewards?
- How should post-training objectives keep a policy near the preferred outcome distribution instead of pushing it into high-score artifacts?
- Can rubrics serve as both human-legible evaluation objects and trainable reward functions?
- How can open-ended post-training preserve quality without collapsing the diversity of valid responses?
- When are embedding distances faithful enough to use as evaluation signals, reward features, or data-selection criteria?
- Where do embedding-space signals help, and where do they hide the mechanism that a sparse feature, probe, or activation-level analysis would expose?
- When is it worth paying the generation cost for generated-token embeddings instead of using ordinary prompt embeddings?
- How robust are generated-token embeddings when a policy is optimized against an embedding-based reward or evaluator?
- Does a post-training run teach a genuinely new reasoning mechanism, or mostly teach a deployment policy over mechanisms learned during pretraining?
- Can reasoning-mechanism rewards improve timing and orchestration without rewarding superficial chain-of-thought style?
- When are natural-language activation explanations reliable enough to inform rubrics, audits, or monitors?
- Which sparse-feature metrics remain meaningful after a model is optimized against feature-derived rewards or monitors?
- When is a worse-reconstructing sparse dictionary more useful because its features are less absorbed, split, or composed?
- How should feature monitors handle multi-level concepts where broad and specific latents both matter?
- When does post-training select among pre-existing personas rather than teaching new non-persona agency from scratch?
- Which persona features are robust enough to audit, steer, or penalize without creating monitor evasion?
- Can hidden same-model continuations reveal failures that are not visible to a separate evaluator reading only the prompt and output?
- How can a second-personality monitor be trained without inheriting or amplifying the main personality's misalignment?
- When should a benchmark score the reasoning process rather than the final answer, especially in pluralistic domains?
- Can rubric-based moral reasoning evaluation become a training signal without rewarding superficial criterion-name-dropping?
- How can flourishing-oriented evaluation avoid becoming paternalistic preference control?
- Which short-term signals best predict long-term user autonomy, competence, relatedness, and growth?
- How should an agent memory benchmark separate helpful institutional context from distracting stale context?
- When should production traces become external memory, fine-tuning data, rubric examples, or ordinary documentation?
- What evidence would show that a feature reward is training the intended behavior rather than training probe evasion?
- How do Scale-style agent/evaluation papers connect to model-level post-training rather than only product evaluation?
