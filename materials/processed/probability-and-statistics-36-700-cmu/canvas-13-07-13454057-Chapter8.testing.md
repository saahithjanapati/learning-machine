## 1 Testing Overview

The typical (and most basic) setting is that we observe:
$$X_1, \dots, X_n \sim f_\theta$$
and want to test if $\theta = \theta_0$ or not. A typical example is where we have a coin and would like to know if the coin is fair or not. In a clinical trial we might have a control group and a group taking the drug, and we would like to know if the difference in some health outcome is 0 or not.

The way we formalize this is by defining a *null hypothesis* $H_0$ and an *alternative hypothesis* $H_1$.

So we would say:
$$H_0 : \theta = \theta_0$$
$$H_1 : \theta \neq \theta_0.$$

The more general case is that we have two sets of parameters $\Theta_0$ and $\Theta_1$ which are non-overlapping, i.e. $\Theta_0 \cap \Theta_1 = \emptyset$ and would like to test the hypothesis:
$$H_0 : \theta \in \Theta_0$$
$$H_1 : \theta \in \Theta_1.$$

We will refer to the case when $\Theta_0$ is a single point as a *simple* null versus the more general case of a *composite* null.

**Example:** In the example above of testing if a coin is fair or not. We have
$$X_1, \dots, X_n \sim \text{Bernoulli}(p).$$
Our null and alternate hypotheses are:
$$H_0 : p = 1/2$$
$$H_1 : p \neq 1/2.$$
In this case we have a simple null and a composite alternative.

**Remark.** In hypothesis testing, the question is never if the null hypothesis is true or not, since we will never know for sure. Rather the question of interest is whether we have sufficient evidence to reject the null hypothesis or not. So in hypothesis testing, there are two possibilities: (1) you reject the null hypothesis or (2) you retain/fail to reject it. To reiterate, retaining the null hypothesis is not equivalent to saying that the null hypothesis is true. We just don’t know what is true. All we can do is take decision that have the properties we want: a stated $\alpha$ level (probability of rejecting the null by mistake), and high power.

**Type I and Type II errors.** There are two types of errors one might make in hypothesis testing: a *Type I error* is when the null hypothesis is true but was incorrectly rejected, and a *Type II error* is when the alternate hypothesis was true but we failed to reject the null.

Usually Type I and Type II errors are treated asymmetrically, as the null and alternative hypotheses are asymmetric. It is often helpful to make the analogy to a court trial: the null hypothesis is that the defendant is innocent, and the alternative is that the defendant is guilty.

### 1.1 Construction of Tests

The typical way we construct tests is:

1. We choose a *test statistic* $T_n = T_n(X_1, \dots, X_n)$, i.e. a function of the data that gives us information about what we want to test.
2. We determine the *rejection region* $R$.
   For example, reject $H_0$ if the value of $T_n > c$ (or $< c$, or whatever is appropriate for the problem), where $c$ is such that the probability of rejecting $H_0$ when $H_0$ is true is capped at (capped at or ideally equal to) some small $\alpha$ that we choose.
3. If $T_n \in R$ we reject $H_0$ otherwise we retain $H_0$.

**Example:** Suppose again $X_1, \dots, X_n \sim \text{Bernoulli}(p)$, and we test:
$$H_0 : p = 1/2$$
$$H_1 : p \neq 1/2.$$
A natural test statistic is an estimator for $p$:
$$T_n = \frac{1}{n} \sum_{i=1}^n X_i,$$
and a natural rejection region is:
$$R = \{(X_1, \dots, X_n) \text{ such that } |T_n(X_1, \dots, X_n) - 1/2| \geq c\}.$$
Effectively we reject $H_0$ if $T_n$ is far from $1/2$. We need to choose $c$ to ensure that the test has good properties.

More generally, we need to choose both the test statistic $T$ and the rejection region $R$ to ensure our tests have desirable/stated properties. Let us now discuss how we evaluate tests.

### 1.2 Evaluating Tests

For simplicity, assume we have a simple null hypothesis $H_0 : \theta = \theta_0$. We reject the null hypothesis when $(X_1, \dots, X_n) \in R$. We define the *power function* as:
$$\beta(\theta) = P_\theta((X_1, \dots, X_n) \in R).$$

That is, the power is the probability of rejecting $H_0$ when the parameter has value $\theta$. We want the rejection region $R$ to be such that
$$\beta(\theta_0) = P_{\theta_0}((X_1, \dots, X_n) \in R) = \alpha.$$
That is, we want the probability of rejecting $H_0$ when the parameter has value $\theta_0$, i.e. when $H_0$ is true, to be $\alpha$. In the class notation,
$$P(\text{reject } H_0 \text{ when } H_0 \text{ is true}) = \alpha.$$
Tests of this form are called level $\alpha$ tests.

**Example:** Suppose $X_1, \dots, X_n \sim N(\theta, \sigma^2)$ with $\sigma^2$ known. We want to test:
$$H_0 : \theta = \theta_0$$
$$H_1 : \theta > \theta_0.$$
The alternative here is called a *one-sided alternative*.

A natural test statistic is the average $\bar{X}_n = \frac{1}{n} \sum_{i=1}^n X_i$ but we will re-scale it for convenience:
$$T_n(X_1, \dots, X_n) = \frac{\bar{X}_n - \theta_0}{\sigma/\sqrt{n}}.$$
and we reject if $T_n > c$ for some threshold (critical value) $c$. The power function is:
$$\beta(\theta) = P_\theta(T_n > c) = P_\theta\left(\frac{\bar{X}_n - \theta}{\sigma/\sqrt{n}} > c + \frac{\theta_0 - \theta}{\sigma/\sqrt{n}}\right) = P\left(Z > c + \frac{\theta_0 - \theta}{\sigma/\sqrt{n}}\right) = 1 - \Phi\left(c + \frac{\theta_0 - \theta}{\sigma/\sqrt{n}}\right),$$
since
$$\frac{\bar{X}_n - \theta}{\sigma/\sqrt{n}} \sim N(0, 1).$$
When $H_0$ is true, i.e. when $\theta = \theta_0$, the power is
$$\beta(\theta_0) = 1 - \Phi(c),$$
which is the probability of making a type I error, and we find $c$ by setting $\beta(\theta_0) = \alpha$. This yields
$$c = \Phi^{-1}(1 - \alpha) = z_\alpha$$

**Example:** Suppose $X_1, \dots, X_n \sim N(\theta, \sigma^2)$ with $\sigma^2$ known. We want to test:
$$H_0 : \theta = \theta_0$$
$$H_1 : \theta \neq \theta_0.$$

This is now a *two-sided alternative*. A natural idea is to reject $H_0$ if $|T_n| > c$ for some threshold $c$. In this case, the power function is:
$$\beta(\theta) = P_\theta(T_n < -c) + P_\theta(T_n > c),$$
which as before we can expand as:
$$\beta(\theta) = P_\theta\left(\frac{\bar{X}_n - \theta}{\sigma/\sqrt{n}} < -c + \frac{\theta_0 - \theta}{\sigma/\sqrt{n}}\right) + P_\theta\left(\frac{\bar{X}_n - \theta}{\sigma/\sqrt{n}} > c + \frac{\theta_0 - \theta}{\sigma/\sqrt{n}}\right)$$
$$= \Phi\left(-c + \frac{\theta_0 - \theta}{\sigma/\sqrt{n}}\right) + 1 - \Phi\left(c + \frac{\theta_0 - \theta}{\sigma/\sqrt{n}}\right).$$
Under the null we have that:
$$\beta(\theta_0) = \Phi(-c) + 1 - \Phi(c) = 2\Phi(-c),$$
which we set to be equal to $\alpha$, which yields the critical value:
$$c = -\Phi^{-1}(\alpha/2) = \Phi^{-1}(1 - \alpha/2) = z_{\alpha/2}$$

To summarize our progress so far: we have seen how to set up a hypothesis testing problem formally. For a given test statistic we set the threshold to ensure that the test has level $\alpha$, i.e. to ensure that the probability of rejecting $H_0$ by mistake is $\alpha$.

This however, pre-supposes that we know how to come up with a good/reasonable test statistic. We will see how to do that using likelihood ratio tests later on. But first let’s do another example and learn about some famous tests.

**Example (Comparing two Bernoullis via a paired comparison test):** A typical scenario is that we have two prediction algorithms that we would like to compare. Suppose first, that we test the two algorithms by evaluating them on two different test sets: Algorithm 1 on a test set of size $m$ and Algorithm 2 on a test set of size $n$. Let $X$ be the number of correct predictions made by Algorithm 1 and $Y$ be the number of correct predictions made by Algorithm 2. Then $X \sim \text{Binomial}(m, p_1)$ and $Y \sim \text{Binomial}(n, p_2)$ where $p_1$ and $p_2$ are the error rates of the first and second Algorithm respectively. Defining $\delta = p_1 - p_2$, our hypotheses are:
$$H_0 : \delta = 0$$
$$H_1 : \delta \neq 0.$$
Let $\hat{\delta} = \hat{p}_1 - \hat{p}_2$. It makes sense to reject the null hypothesis if $|\hat{\delta}| > c$. We estimate the variance of our estimator in the usual way:
$$\text{Var}(\hat{\delta}) = \hat{p}(1 - \hat{p})\left(\frac{1}{n} + \frac{1}{m}\right),$$

In this case the two predictions are not independent so we cannot directly compute the variance of $\hat{\delta}$. You can try to do this and observe that it will depend on the covariance between the two algorithms which you will have to estimate.

A convenient way to test this hypothesis is via a paired comparison test. The high-level principle is to try to pair outcomes in a certain way. Typically this is done in randomized studies, for instance via paired t-tests, in order to eliminate certain biases. In this case, one can imagine testing both Algorithms on the data but recording, the difference $D_i = X_i - Y_i$, i.e.

1. $D_i = 0$ if both algorithms have the same prediction.
2. $D_i = 1$ if Algorithm 1 is correct while 2 is wrong.
3. $D_i = -1$ if Algorithm 2 is correct while 1 is wrong.

In this case you can observe that:
$$\delta = E(X) - E(Y) = E(X - Y) = E(D).$$
A natural test statistic is an estimator for $\delta$:
$$T_n = \frac{\frac{1}{n} \sum_{i=1}^n D_i}{\sqrt{\text{Var}(\frac{1}{n} \sum_{i=1}^n D_i)}},$$
and the variance is:
$$\text{Var}\left(\frac{1}{n} \sum_{i=1}^n D_i\right) = \frac{1}{n^2} \sum_{i=1}^n \left(D_i - \frac{1}{n} \sum_{i=1}^n D_i\right)^2.$$
We thus reject the null hypothesis if:
$$|T_n| \geq \Phi^{-1}(1 - \alpha/2).$$

### Evaluating Tests – Take 2

Below is how tests are introduced in more generality, when the null hypothesis is composite.

Suppose that we reject the null hypothesis when $(X_1, \dots, X_n) \in R$. We define the *power function* as:
$$\beta(\theta) = P_\theta((X_1, \dots, X_n) \in R).$$

We would like that $\beta(\theta)$ to be small over $\Theta_0$ and large over $\Theta_1$. The Neyman-Pearson paradigm is the following:

1. Pick an $\alpha \in [0, 1]$.
2. Then try to maximize $\beta(\theta)$ over $\Theta_1$ subject to
$$\sup_{\theta \in \Theta_0} \beta(\theta) \leq \alpha.$$

Tests of this form are called level $\alpha$ tests, i.e. level $\alpha$ tests are ones for which: $\sup_{\theta \in \Theta_0} \beta(\theta) \leq \alpha$. Lets look at a the same couple of examples as above.

**Example:** Suppose $X_1, \dots, X_n \sim N(\theta, \sigma^2)$ with $\sigma^2$ known. We want to test:
$$H_0 : \theta = \theta_0$$
$$H_1 : \theta > \theta_0.$$
As above, we reject $H_0$ if $T_n = \frac{\bar{X}_n - \theta_0}{\sigma/\sqrt{n}} > c$ for some $c$. The power function is:
$$\beta(\theta) = 1 - \Phi\left(c + \frac{\theta_0 - \theta}{\sigma/\sqrt{n}}\right),$$
as derived earlier. Now we try to implement the Neyman-Pearson paradigm. We want to pick the threshold $c$ so that:
$$\sup_{\theta \in \Theta_0} \left[ 1 - \Phi\left(c + \frac{\theta_0 - \theta}{\sigma/\sqrt{n}}\right) \right] \leq \alpha,$$
which is the same as:
$$1 - \Phi(c) \leq \alpha,$$
We want to maximize $\beta(\theta)$ when $\theta > \theta_0$ so we use the threshold:
$$c = \Phi^{-1}(1 - \alpha) = z_\alpha$$

## 2 P-values

A p-value is a way to try to extract more information than “reject” or “fail to reject” in the hypothesis testing framework.

The p-value is the smallest $\alpha$ for which the test would reject the null hypothesis. Intuitively, a smaller p-value is stronger evidence against the null. Scientists often report p-values, and informally a p-value of $< 0.01$ is considered strong evidence against the null, and $< 0.05$ is moderate evidence against the null.

For example, suppose that we want to test a the simple null $H_0 : \theta = \theta_0$ and that the test is one-sided and the size $\alpha$ test is of the form: reject $H_0$ if
$$T_n \geq c_\alpha$$
for some threshold $c_\alpha$. Then the p-value is given by:
$$\text{p-value} = P_{\theta_0}(T(X_n) \geq T_n) = P(T(X_n) \geq T_n \text{ when } H_0 \text{ is true}),$$
which is often easy to compute.

The above expression also gives a common interpretation of a p-value. A p-value is just the probability under the null of seeing a more (or equally) extreme test statistic than the one observed.

It is easy to extend this reasoning to two-sided tests. Say that we have a simple null $\theta = \theta_0$, and our test rejects at level $\alpha$ if $|T_n| \geq z_{\alpha/2}$.

Suppose that $T_n$ is positive, then the probability of seeing a test statistic larger that $T_n$ is just $1 - \Phi(T_n)$ and the probability of seeing a test statistic smaller than $-T_n$ is $\Phi(-T_n) = 1 - \Phi(T_n)$. In this case the p-value:
$$\text{p-value} = 2(1 - \Phi(T_n)) = 2\Phi(-T_n).$$

More generally, when we do not assume $T_n$ is positive this is given by:
$$\text{p-value} = 2\Phi(-|T_n|).$$

**Remark 1:**
> A p-value is a random variable, because it is a function of the data.

Intuitively, a smaller p-value is stronger evidence against the null. Scientists often report p-values, and informally a p-value of $< 0.01$ is considered strong evidence against the null, and $< 0.05$ is moderate evidence against the null.

**Remark 2:**
> p-value IS NOT $\mathbb{P}(H_0 \text{ is true})$.

The latter probability is not well-defined since $\theta$ is not random (p-values are not used in a Bayesian framework).

### 3.1 Pearson’s $\chi^2$ test
So far we have done tests about a scalar parameter. We are sometimes interested in vector parameters.

Pearson’s $\chi^2$ test was originally used to test hypotheses about multinomials. More generally, one can use it to test hypotheses about multivariate parameters that have a Gaussian distribution (at least asymptotically) under the null.

#### The $\chi^2$ distribution
A reminder: suppose we have $Z_1, \dots, Z_k$ which are independent standard Gaussian RVs. Then the variable:
$$V_k = \sum_{i=1}^k Z_i^2,$$
has a $\chi^2$ distribution with $k$ degrees of freedom. It is easy to check that the mean $\mathbb{E}[V_k] = k$, with some effort you can also show that the variance $\text{Var}(V_k) = 2k$.

We will use $\chi^2_{k,\alpha}$ to denote the upper $\alpha$ quantile of this distribution, i.e.
$$\mathbb{P}(V_k \geq \chi^2_{k,\alpha}) = \alpha.$$

#### Testing a vector of Gaussian means
Let us consider a different *multivariate* example. Suppose we observe $X_1, \dots, X_n \sim N(\theta, I_d)$, where $\theta \in \mathbb{R}^d$, and $I_d$ is the $d \times d$ identity matrix. We would like to test the hypotheses:
$$H_0 : \theta = \theta_0$$
$$H_1 : \theta \neq \theta_0.$$

In this case, we could first form:
$$\hat{\theta} = \frac{1}{n} \sum_{i=1}^n X_i,$$
and observe that under the null:
$$\sqrt{n}(\hat{\theta} - \theta_0) \sim N(0, I_d).$$

$$T = n \sum_{j=1}^d (\hat{\theta}_j - \theta_{0j})^2,$$

$$\mathbb{P}(\chi^2_d \geq \chi^2_{d,\alpha}) = \alpha.$$

#### Testing multinomials
Suppose that we observe counts $(X_1, \dots, X_k)$ for $n$ samples drawn from a multinomial on $k$ categories with probabilities $(p_1, \dots, p_k)$. The MLE is given by:
$$\hat{p}_i = X_i/n.$$

For some fixed vector of probabilities $p_0$, we want to test the hypotheses:
$$H_0 : p = p_0$$
$$H_1 : p \neq p_0.$$

In class, we came up with various test statistics that made sense, e.g.
$$T_1 = \sum_{i=1}^k |\hat{p}_i - p_{0i}|^\gamma, \quad \gamma > 0,$$
$$T_2 = \mathbb{V}(\hat{p}_i, i = 1, \dots, k),$$
$$T_3 = \max_i |\hat{p}_i - p_{0i}|.$$

These all make sense, but we have two problems: do they yield powerful tests, and how do we derive their null distributions? (the answer to the last question is: it is hard, unless we use simulation methods.)

Pearson suggested the test statistic:
$$T = \sum_{i=1}^k \frac{(X_i - np_{0i})^2}{np_{0i}} = \sum_{i=1}^k \frac{(\hat{p}_i - p_{0i})^2}{p_{0i}}.$$

Here $X_i$ is the observed count, and $np_{0j}$ expected count so the statistic makes intuitive sense: we expect it to be large under the alternate and small under the null. The precise reason for this form actually comes from the central limit theorem. It is not difficult to show that:
$$\frac{(X_i - np_{0i})}{\sqrt{np_{0i}}} \to N(0, 1),$$
in distribution under the null hypothesis. This is nice because the test statistic $T$ then just has a $\chi^2$ distribution with $k - 1$ degrees of freedom under the null. There are only $k - 1$ dof because there are really only $k - 1$ parameters, not $k$, since $\sum_i p_i = 1$.

In this case, we would reject the null hypothesis if:
$$T \geq \chi^2_{k-1,\alpha}.$$

The Pearson test can be used in any case where you have an appropriate central limit theorem for the parameter estimates.

**Example: Mendel’s experiment.** Mendel’s theory predicts that the probability of a pea plant in a certain hybrid generation falling in one of four categories is $\vec{p}_0 = (9/16, 3/16, 3/16, 1/16)$. The observed count vector is $X = (315, 101, 108, 32)$ with a total count $n = 556$. The predicted counts are $\vec{E}_0 = n\vec{p}_0 = (312.75, 104.25, 104.25, 34.75)$. The $\chi^2$ statistic is
$$T = \frac{(315 - 312.75)^2}{312.75} + \frac{(101 - 104.25)^2}{104.25} + \frac{(108 - 104.25)^2}{104.25} + \frac{(32 - 34.75)^2}{34.75} = 0.47.$$
The 0.05 upper quantile of $\chi^2_3$ is 7.815. So we retain the null, and we conclude that the collected data does not invalidate Mendel’s hypothesis.

This test is an example of a goodness-of-fit test. When Mendel performed that test, he was hoping that $H_0$ would NOT be rejected, because he was hoping that his theory was correct. And indeed, $H_0$ was not rejected at the 5% significance level. The problem with tests that hope to retain the null hypothesis is that the type II error is not controlled and could be quite large.

There are many more tests that end up being $\chi^2_d$ tests, including tests for independence of factors in contingency tables. They all rely on a test statistics being the distance between a set of observed numbers and the numbers we would expect if the null hypothesis was true.

Given two random variables $X$ and $Y$, it is often of interest to understand if the two random variables are dependent, and to estimate the strength of their dependence.

Concretely, we would like to study the hypothesis testing question:
$$H_0 : X \perp Y$$
$$H_1 : X \not\perp Y,$$
where we use the symbols $\perp$ and $\not\perp$ to denote independent and not independent respectively.

Broadly, one of the key ideas is to use the fact that if $X$ and $Y$ are independent then their joint distribution will be equal to the product of their marginals. We can use the “distance” between the joint and product of marginals as a measure of association.

#### Two binary variables
To begin with let us consider the simplest possible setting. Suppose that $(X, Y)$ are two binary random variables and we observe $n$ i.i.d pairs $(X_1, Y_1), \dots, (X_n, Y_n)$.

We can represent this binary data in a two-by-two table, i.e. a table of the form:

| | Y = 0 | Y = 1 | |
| :--- | :---: | :---: | :---: |
| **X = 0** | $C_{00}$ | $C_{01}$ | $C_{0\cdot}$ |

With all of this in place we can define our first test for independence. This is the Pearson $\chi^2$ test. The test statistic is:
$$T = \sum_{i=0}^1 \sum_{j=0}^1 \frac{(C_{ij} - E_{ij})^2}{E_{ij}},$$

$$T \geq \chi^2_{1,\alpha}.$$

### 3.3 Permutation test
The next important test we will discuss is a non-parametric two-sample test. In the two-sample testing setting we observe:

$$H_0 : F_X = F_Y$$
$$H_1 : F_X \neq F_Y.$$

There are various possible test statistics. For example we could look at some distance measure between of EDFs of $X$ and $Y$. But a simpler, and common one is to use a difference in means:
$$T(X_1, \dots, X_m, Y_1, \dots, Y_n) = \left| \frac{1}{m} \sum_{i=1}^m X_i - \frac{1}{n} \sum_{i=1}^n Y_i \right|,$$
one could also standardize this statistic by its variance, or consider more complex test statistics based on signs and ranks. Let us denote the test statistic computed on the data we observed as $T_{obs}$.

In general, since we have not assumed anything about $F_X$ and $F_Y$ it is not easy to compute the distribution of our test statistic, and approximations (based on a CLT for instance) might be quite bad. The permutation test, gives a way to design an *exact* $\alpha$ level test without making any approximations.

The idea of the permutation test is simple. Define $N = m + n$ and consider all $N!$ permutations of the data $\{X_1, \dots, X_m, Y_1, \dots, Y_n\}$. For each permutation we could compute our test statistic $T$. Denote these as $T_1, \dots, T_{N!}$.

The key observation is: **under the null hypothesis each value $T_1, \dots, T_{N!}$ has the same distribution (even if we do not know what it is).**

Suppose we reject for large values of $T$. Then we could simply define the p-value as:
$$\text{p-value} = \frac{1}{N!} \sum_{i=1}^{N!} \mathbb{I}(T_i > T_{obs}).$$

It is important to note that this is an exact p-value whatever the sample sizes, i.e. no asymptotic approximations are needed to show that rejecting the null when this p-value is less than $\alpha$ controls the Type I error at $\alpha$. And that is true even for tiny sample sizes (see next example). (If you want to check this out, look at conditional tests. Here, the EDF of the data under $H_0$ is a sufficient statistic for $F$, and the test we do is conditioned on it, so the test does not actually depend on the unknown $F$.)

> **Algorithm for Permutation Test**
> 1. Compute the observed value of the test statistic
>    $t_{obs} = T(X_1, \dots, X_m, Y_1, \dots, Y_n)$.
> 2. Randomly permute the data. Compute the statistic again using the permuted data.
> 3. Repeat the previous step $B$ times and let $T_1, \dots, T_B$ denote the resulting values.
> 4. The approximate p-value is
>    $$\frac{1}{B} \sum_{j=1}^B I(T_j > t_{obs}).$$

# 4 Systematic methods to choose good test statistics

So far we have used our intuition to find test statistics (TS). There are systematic ways to do that, which yield "good" TS, i.e. TS that yield tests with optimal, or at least very good power.

We start with the Wald test, which we have encountered several times already. It is applied when the parameter to best tested is a scalar. When the parameter is multi-dimensional, we use the general likelihood ratio; this test is often equivalent to the Wald test, when the parameter is a scalar. We end with a general method to construct the *optimal* test for particular simple hypothesis testing problems. This test is called the Neyman-Pearson test or the likelihood ratio test. It is interesting because we can prove it is optimal.

### 4.1 The Wald Test

The Wald test considers hypothesis testing problems of the form:
$$H_0 : \theta = \theta_0$$
$$H_1 : \theta \neq \theta_0,$$
where $\theta$ is a scalar parameter. Wald basically suggested that we use an estimator that is asymptotically normal (at least under the null), i.e. for example we could use the MLE. We have seen that the MLE typically (given regularity conditions) satisfies:
$$\sqrt{nI(\theta)}(\hat{\theta} - \theta) \to N(0, 1),$$
in distribution as $n \to \infty$. Under the null hypothesis the MLE typically satisfies:
$$\sqrt{nI(\theta_0)}(\hat{\theta} - \theta_0) \to N(0, 1),$$
in distribution as $n \to \infty$. So an idea would be to use the standardized MLE as our test statistic, i.e.:
$$T_n = \sqrt{nI(\theta_0)}(\hat{\theta} - \theta_0),$$
and then construct the rejection threshold the same way we did before. That is, we reject the null hypothesis if:
$$|T_n| \geq \Phi^{-1} \left( 1 - \frac{\alpha}{2} \right),$$
for the two sided alternative hypothesis stated above. It is easy to see that asymptotically this is a correct size-$\alpha$ test.

**Example:** Suppose that we were testing the null $H_0 : p = p_0$ in a Bernoulli problem. We would use:
$$T_n = \frac{\hat{p} - p_0}{\sqrt{p_0(1 - p_0)/n}},$$
and reject the null if $|T_n| \geq \Phi^{-1} \left( 1 - \frac{\alpha}{2} \right)$.

**Other examples:** All the examples we have done so far where we tested a scalar $\theta$ were Wald tests (and generalized likelihood ratio tests; see later). Check it out.

### 4.2 The General Likelihood Ratio test

As we have seen with some examples in Section 2, it is not as obvious to come up with TS, or with TS that we know yield powerful tests when $\theta$ is a vector of parameters. The general LRT is a systematic way of finding TS with good properties.

Recall that if $X_1, \dots, X_n$ is a random sample for a population with pdf or pmf $f(x|\theta)$ ($\theta$ may be a vector), the likelihood function is
$$L(\theta|x_1, \dots, x_n) = L(\theta|\mathbf{x}) = f(\mathbf{x}|\theta) = \prod_{i=1}^n f(x_i|\theta).$$
Let $\Theta$ denote the entire parameter space. The *likelihood ratio test statistic* for testing $H_0 : \theta \in \Theta_0$ versus $H_1 : \theta \in \Theta_0^c$ is
$$\lambda(\mathbf{x}) = \frac{\sup_{\Theta_0} L(\theta|\mathbf{x})}{\sup_{\Theta} L(\theta|\mathbf{x})} = \frac{L(\hat{\theta}_0|\mathbf{x})}{L(\hat{\theta}|\mathbf{x})},$$
where $\hat{\theta}$ is the MLE obtained by doing an unrestricted maximization of $L(\theta|\mathbf{x})$, and $\hat{\theta}_0$ is obtained by maximizing the likelihood function assuming $\Theta_0$ is the parameter space. The rejection region is $RR = \{\mathbf{x} : \lambda(\mathbf{x}) \leq c\}$, where $c \in [0, 1]$ is such that
$$\sup_{\theta \in \Theta_0} P_\theta(\lambda(\mathbf{X}) \leq c) \leq \alpha, \quad (1)$$

Now the LRT test not only gives us a TS, but also an asymptotic null distribution, so we can calculate the critical value $c$. Indeed, under some regularity conditions on the model $f(x|\theta)$ (existence and behavior of the derivatives wrt the parameter of the likelihood, and support of the distribution (it cannot depend on the parameter)), and if $f(x|\theta)$ under $H_0$ is a nested model of $f(x|\theta)$ under $H_A$, then the distribution of $T = -2 \log \lambda(\mathbf{X})$ converges to a chi squared distribution **when $H_0$ is true**, as $n \to \infty$.

Rejection of $H_0$ for small values of $\lambda(\mathbf{X})$ is equivalent to rejection for large values of $-2 \log \lambda(\mathbf{X})$. Thus,
$$H_0 \text{ is rejected if and only if } - 2 \log \lambda(\mathbf{X}) \geq \chi^2_{\nu,\alpha},$$
where $\nu$, the degrees of freedom of the limiting distribution, is the difference between the number of free parameters specified by $\theta \in \Theta_0$ and the number of free parameters specified by $\theta \in \Theta$. **Example:** Let $X_1, \dots, X_n$ be i.i.d $N(\theta, 1)$ random variables. We want to test $H_0 : \theta = \theta_0$ versus $H_1 : \theta \neq \theta_0$. Here, $\theta$ is a scalar so we could do a Wald test, but let's try the LRT.

Since there is only one value of $\theta$ specified by $H_0$, the numerator of $\lambda(\mathbf{x})$ is $L(\theta_0|\mathbf{x})$. The (unrestricted) MLE of $\theta$ is $\bar{X}$, the sample mean. Therefore the LR statistic is
$$\lambda(\mathbf{x}) = \frac{(2\pi)^{-n/2} \exp[-\sum_{i=1}^n (x_i - \theta_0)^2/2]}{(2\pi)^{-n/2} \exp[-\sum_{i=1}^n (x_i - \bar{x})^2/2]} \quad (2)$$
$$= \exp \left[ -\frac{1}{2} \left( \sum_{i=1}^n (x_i - \theta_0)^2 - \sum_{i=1}^n (x_i - \bar{x})^2 \right) \right].$$
and we reject $H_0$ for (small values of $\lambda(\mathbf{x})$, or equivalently for) large values of $-2 \log \lambda(\mathbf{X})$. That is, the rejection region is
$$\mathbf{x} : -2 \log \lambda(\mathbf{X}) \geq c, \text{ where } c = \chi^2_{\nu,\alpha}, \text{ and } \nu = 1.$$

**The Neyman-Pearson Lemma**

The Neyman-Pearson Lemma says that the NP test is the most powerful test of size $\alpha$. This means that if we have *any other test* that controls the Type I error rate at $\alpha$, then its power is at most the power of the NP test.

**Proof:** First, recall that the **power of a test** is the probability of correctly rejecting the null, i.e., the probability of rejecting the null when the alternative is true. The **size of a test** is the probability of incorrectly rejecting the null – this is the quantity we usually control at $\alpha$.

We will associate every test with an indicator function $\phi$. Recall, that $R$ is the rejection region for our test, i.e. if $x \in R$ we reject the null and otherwise we accept it. Then:
$$\phi(x) = \begin{cases} 1, & \text{if } x \in R \\ 0, & \text{if } x \notin R. \end{cases}$$
So we can write the power of a test as:
$$\int_x \phi(x)f_1(x)dx \quad \text{and the size as} \quad \int_x \phi(x)f_0(x)dx,$$
using the law of the lazy statistician and remembering that the expectation of an indicator function is a probability (see Chapter 3).

Let us denote the test function of the NP test as $\phi_{NP}$ and the test function of any other test we want to compare against as $\phi_A$.

$$\int_x \underbrace{(\phi_{NP}(x) - \phi_A(x))}_{T_1} \underbrace{\left( f_1(x) - \frac{f_0(x)}{c^*} \right)}_{T_2} dx \geq 0,$$
which will imply that
$$\int_x (\phi_{NP}(x) - \phi_A(x))f_1(x)dx \geq \frac{1}{c^*} \int_x (\phi_{NP}(x) - \phi_A(x))f_0(x)dx$$
$$= \frac{1}{c^*} \left( \underbrace{\int_x \phi_{NP}(x)f_0(x)dx}_{=\alpha} - \underbrace{\int_x \phi_A(x)f_0(x)dx}_{\leq \alpha} \right)$$
$$\geq 0.$$
that is, the power of NP test is higher than the power of test A, since the difference of their powers is negative.

To see this we can just consider all possible cases:

1. If we see a data sample $x$ such that both tests reject or both tests accept, then $T_1 = 1 - 1 = 0$ or $T_1 = 0 - 0 = 0$ so the inequality is clearly true since the LHS is 0.
2. If we see a data sample $x$ such that NP rejects, and the test A accepts then $\phi_{NP}(x) = 1$, and $\phi_A(x) = 0$, so $T_1 \geq 0$. Since the NP test rejected the null we know that:
   $$\frac{f_0(x)}{f_1(x)} \leq c^*,$$
   so that $T_2 \geq 0$. So the inequality is true in this case.
3. If NP accepts and the test A rejects then both $T_1$ and $T_2$ are negative so the inequality is also true in this case.

So we can see that for every $x$, $T_1 \times T_2 \geq 0$ so it is true when we integrate over $x$, which proves what we need.

# 5 Multiple Testing

The problem of multiple testing is one that is fundamental to a lot of science. Typical modern scientific discovery does not always proceed in a simple fashion where we have a single hypothesis that we would like to test.

A classical example is in the analysis of gene expression data. We measure the expression of tens of thousands of genes and we would like to know if any of them are associated with some phenotype (for example whether a person has a disease or not). Typically, the way this is done is that the scientist does tens of thousands of hypothesis tests, and then reports the associations that are significant, i.e. reports the tests where the null hypothesis was rejected.

This is very problematic:

Suppose we did 1000 hypothesis tests, and for each of them rejected the null when the p-value was less than $\alpha = 0.05$. How many times would you expect to falsely reject the null hypothesis?

The answer is we would expect to reject the null hypothesis 50 times. So we really cannot report all the discovered associations (rejections) as significant because we expect many false rejections.

The multiple testing problem is behind a lot of the “reproducibility crisis” of modern science. Many results that have been reported significant cannot be reproduced simply because they are false rejections. Too many false rejections come from doing multiple testing but not properly adjusting your tests to reflect the fact that many hypothesis tests are being done$^1$.

The basic question is how to we adjust our p-value cutoffs to account for the fact that multiple tests are being done.

### 5.1 The Family-Wise Error Rate

Recall that the Type I error controls the probability of falsely rejecting a null hypothesis. But when many tests are done, perhaps it makes more sense to control the probability of falsely rejecting *any* (i.e. at least one) null hypothesis. This is called the Family-Wise Error Rate (FWER). The FWER is the probability of falsely rejecting at least one null hypothesis once amongst the multiple tests. The basic question is then: how do we control the FWER?

#### Sidak correction

Recall that to control the Type I error of a test, we reject the null if the p-value $\leq \alpha$, and doing this controls the Type I error at $\alpha$, for example $\alpha = 1\%$.

Now suppose we do $p$ hypothesis tests, and we want to control the FWER at $\beta$, for example $\beta = 5\%$. The Sidak correction says we should reject any test if its p-value is smaller than:
$$\text{p-value} \leq 1 - (1 - \beta)^{1/p} = \alpha,$$
so we reject any test if its p-value is less than $\alpha$. Another way to say this is: use significance level $\alpha = 1 - (1 - \beta)^{1/p}$ for each of the tests. Then if the p-values are independent, this guarantees that $\text{FWER} \leq \beta$.

**Proof:** Suppose that all the null hypotheses are true (this is called the *global null*). The probability of falsely rejecting one test is $\alpha$, so we correctly fail to reject it with probability $1 - \alpha$. Since the p-values are all independent the probability of falsely rejecting any null hypothesis is:
$$\text{FWER} = \mathbb{P}(\text{falsely reject at least 1 null}) = 1 - \mathbb{P}(\text{reject NO true null}) = 1 - (1 - \alpha)^p = \beta,$$
which yields $\alpha = 1 - (1 - \beta)^{1/p}$. If only $p_0$ out of the $p$ tests have $H_0$ true, we should use $\alpha = 1 - (1 - \beta)^{1/p_0}$ to get $\text{FWER} = \beta$, but we still use $\alpha = 1 - (1 - \beta)^{1/p}$ since we don’t know $p_0$. Then $\text{FWER} = 1 - (1 - \alpha)^{p_0} = 1 - ((1 - \beta)^{1/p})^{p_0} = 1 - (1 - \beta)^{p_0/p} \leq \beta$ (instead of $= \beta$) since $p_0 \leq p$. This makes the test unnecessarily conservative but there is no way around this.

#### Bonferroni correction

The Sidak correction requires independent of p-values. This is unrealistic especially if you compute the test statistics for the different tests on the same set of data. The Bonferroni correction instead uses the union bound to avoid this assumption.

The Bonferroni correction says we reject any test if the p-value is smaller than:
$$\text{p-value} \leq \beta/p.$$

Another way to say this is: use level $\alpha = \beta/p$ for each of the tests. Then this guarantees $\text{FWER} \leq \beta$.

**Proof:** Suppose again that the global null is true. In this case,
$$\text{FWER} = \mathbb{P}(\text{reject at least 1 null}) = \mathbb{P}\left(\bigcup_{i=1}^p \text{reject } H_{0i}\right) \leq \sum_{i=1}^p \mathbb{P}(\text{reject } H_{0i}) = \sum_{i=1}^p \frac{\beta}{p} = \beta,$$
where the inequality follows from the union bound. Now if only $p_0$ out of the $p$ tests have $H_0$ true, we still use $p$, which implies $\text{FWER} = \sum_{i=1}^{p_0} \frac{\beta}{p} \leq \beta$.

### 5.2 p-value fishing (a.k.a p-hacking) and commercial fishing

We have discussed why and how one must take care when dealing with multiple hypothesis tests and the correct interpretation of the individual p-values. The practice of reporting individual p-values as significant without taking account of multiplicity is called “p-value fishing”. IMO: In principle it would not be a problem if everyone understood what p-values are, but alas this is not the case.

There is another form of p-value fishing that is more insidious. Say scientists have a hypothesis and a dataset. They test their hypothesis and fail to reject $H_0$. They are disappointed, so now they start doing many other tests to find something that is significant so they feel better about having invested so much effort in their experiment. And they will find something for sure, if they keep digging, and they will only report that one significant result without mentioning how many tests they did. The problem is that they don’t adjust the null distribution of the TS, and while they think that the test level of their discovery is $\alpha$, in fact it is a lot higher.

You can think about this in 2 ways: imagine that there is no signal in the data. Then if you perform many independent tests and pick as significant the test(s) with the smallest p-value, what is the distribution of that p-value? Uniform as it should be? (No.) Alternatively, we can use the Sidak reasoning to calculate that the probability of finding at least one significant test is $1 - (1 - \alpha)^p$. If you perform 100 tests at $\alpha = 5\%$, then you will find at least one significant result with a shockingly high probability of 99.4%.

# 6 The False Discovery Rate

The Bonferroni and Sidak corrections can be very conservative (i.e. fail to reject many tests that have $H_0$ false). They ensure that we make at most one single false rejection. In scientific experiments it often makes more sense to control the **False Discovery Rate (FDR)**. The FDR is the expected number of false rejections divided by the number of rejections.

Denote the number of false rejections as $V$, and the total number of rejections as $R$. Then the *false discovery proportion* is:
$$\text{FDP} = \begin{cases} \frac{V}{R} & \text{if } R > 0 \\ 0 & \text{if } R = 0. \end{cases}$$

The FDR is then defined as:
$$\text{FDR} = \mathbb{E}[\text{FDP}].$$

In this notation we can see that the FWER is:
$$\text{FWER} = \mathbb{P}(V \geq 1).$$

We will next consider how one can control the FDR. We will describe a procedure known as the Benjamini-Hochberg (BH) procedure. We will not prove its correctness (its not difficult but it is a bit involved).

### 6.1 The BH procedure

The BH procedure is one that controls the FDR under independence (i.e. the p-values are independent). There is a much weaker form of this procedure that works under dependence (see the Wasserman book). It turns out to be very challenging to tightly control FDR under strong dependence.

The procedure is:

1. Suppose we do $T$ tests. Let us take the p-values $p_1, \dots, p_T$, and sort them, i.e. we create the list: $p_{(1)} \leq p_{(2)} \leq \dots \leq p_{(T)}$.

2. Define the thresholds:
$$t_i = \alpha \frac{i}{T}.$$
We use a different threshold for each of the p-values, instead of the same threshold like we have done so far.

3. Find the largest $i_{\max}$ such that
$$i_{\max} = \arg \max_i \{i : p_{(i)} < t_i\}.$$

4. Reject all nulls upto and including $i_{\max}$.

### 6.2 Properties of FDR

We have now see a procedure that controls the FDR under some assumptions. One question of interest is how does FDR control compare to FWER control? Another is just: how do we interpret FDR control?

**Interpreting FDR control:** The way to think about FDR control is: if we repeat our experiment many times, on average we control the FDP. This is not a statement about the individual experiment we did conduct, and really it does not say much about how likely it is that on a given experiment we have an FDP that is larger than a threshold (think about using Markov’s inequality).

FWER on the other hand, does control the error rate for a single experiment. That is, with FWER control, we have managed our false discoveries unless we are very unlucky; with FDR control, on average our test will control FDP, but in our particular experiment we may not have done a very good job. We will see in a second controlling FWER does control the FDR. The way to interpret all of this is that: FDR control is a very weak notion of error control.

**Connection to FWER:**

1. The first connection is that under the global null (when all the null hypotheses are true) FDR control is equivalent to FWER control.
**Proof:** Under the global null, any rejection is a false rejection. There are two possibilities: either we do not reject anything ($V = 0$): in this case the $\text{FDP} = 0$. If we do reject any null hypothesis ($V \geq 1$) then our $\text{FDP}$ is 1 (since $V = R$). So we have that:
$$\text{FDR} = \mathbb{E}[\text{FDP}] = \mathbb{P}(V > 0) \cdot 1 + \mathbb{P}(V = 0) \cdot 0 = \mathbb{P}(V > 0) = \mathbb{P}(\text{reject at least 1 null}) = \text{FWER}.$$

2. The second connection is that the $\text{FWER} \geq \text{FDR}$ always. This implies that controlling the FWER implies FDR control.
**Proof:** We can see that the following is a simple upper bound on the FDP:
$$\text{FDP} \leq \mathbb{I}(V \geq 1),$$
since if $V = 0$, $\text{FDP} = 0$, and if $V > 0$ then $V/R \leq 1$. Taking expectations of this expression gives:
$$\text{FDR} \leq \mathbb{P}(V \geq 1) = \text{FWER}.$$

The flip-side of this is that FDR control is less stringent so if this is the correct measure for you then you will have *more* power by controlling FDR (rather than controlling FWER).
