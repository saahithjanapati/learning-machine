**1) PAC learning definition [5 points]**

Consider the i.i.d. assumption, the realizable setting, and the 0-1 loss. Write down the definition of PAC learnability of a given hypothesis class $\mathcal{H}$.

[Learning objective: This question just makes sure you know the basic definition. If you do, then this is a freebie :)]

<span style="color:blue">A learning problem $(\mathcal{X}, \mathcal{Y}, \ell)$ is PAC (Probably Approximately Correct) learnable with respect to the hypothesis class $\mathcal{H}$ under the realizability assumption, if there exists a function $n : [0, 1]^2 \to \mathbb{R}$ and an algorithm with the following property: For every $\epsilon, \delta > 0$, for every distribution $\mathcal{D}$ over $\mathcal{X} \times \mathcal{Y}$ running the algorithm on $n(\epsilon, \delta)$ i.i.d. examples generated by $\mathcal{D}$, the algorithm returns a hypothesis $h \in \mathcal{H}$ such that, with probability at least $1 - \delta$, $\text{err}(h) < \epsilon$.</span>

**2) Choosing $C$ in soft-SVM [5+5=10 points]**

Consider soft-SVM (primal form). Recall that there is a parameter $C$ in the objective that multiplies the slack terms $\sum_{i \in [n]} \xi_i$. What happens if $C = 0$? What if $C < 0$? Please justify your answers.

[Learning objective: To help you understand what can go wrong if an inappropriate choice is made.]

<span style="color:blue">If $C = 0$, then the optimization problem assumes a trivial solution, as there is no penalty on the slack variables anymore. The optimization objective reduces to $\|w\|_2^2$, which can be minimized with $w = 0$. In order to satisfy the constraints, the slack variables $\xi_i$ can then be set arbitrarily large. Intuitively, while an infinitely large $C$ corresponds to the hard SVM (i.e., no points allowed inside the margin), an SVM with $C = 0$ allows "all points" to be inside the margin.</span>

<span style="color:blue">If $C < 0$, then the objective can be driven to $-\infty$ increasing the value of the $\xi_i$ variables, and the problem becomes meaningless.</span>

**3) Known distribution [5+5+5=15 points]**

Consider the i.i.d. assumption, 0-1 loss, but there may not be any "true classifier" $h^*$. Instead, suppose you know the distribution $D$ on $\mathcal{X} \times \mathcal{Y}$ from which all $(x, y)$-pairs are drawn. Measure the risk of any hypothesis $h$ as $\mathbb{E}_{(x,y) \sim D}[\mathbb{1}\{h(x) \neq y\}]$. You are given $n$ training data points $\{(x_i, y_i)\}_{i \in [n]}$. Suppose you have (uncountably) infinite computation and storage: you have access to a magical optimization package that will solve any optimization problem that you give it, and furthermore, assume you can store and display any amount of data. Consider the following three questions, and please justify your answer for each of them. For parts a and b below, feel free to write your answers simply in the form of an optimization problem (i.e., as argmin blah subject to blah).

(a) Present a classification algorithm to predict the label of any new point such that the risk is as small as possible.
<span style="color:blue">Given that we know the distribution $D$ from which the data is generated, we can simply predict the value of $y$ that maximizes $p(y|x)$ for the given $x$.
Let $\ell(\hat{y}(x), y)$ be the 0-1 error function for a prediction $\hat{y}(x)$, then the true risk is given by
$$\mathbb{E}_D[\ell(\hat{y}(x), y)] = \iint_{\mathcal{X} \times \mathcal{Y}} \ell(\hat{y}(x), y)p(y|x)p(x)dxdy.$$
Since $y$ can only assume values in $\{-1, 1\}$, $p(y|x)$ assumes only two possible values for any particular $x$. Thus, for any given $x$, choosing $\hat{y}(x)$ as the value of $y$ with higher probability minimizes the term inside the integral, leading to lower risk. Thus, for any given $x$, we can simply predict $\hat{y}(x) = \text{argmax}_y p(y|x)$.</span>

(b) Suppose you are given a hypothesis class $\mathcal{H}$ and are required to output a hypothesis in this class. Now present a classification algorithm under this restriction.
<span style="color:blue">Again, given we know the true distribution $D$, and we have a "magical" optimization algorithm, we can find the optimal model in our hypothesis class that minimizes the true risk (true expected loss) over the data distribution.
In this case, similarly to (a), we are solving the optimization problem
$$\text{arg min}_{h \in \mathcal{H}} \mathbb{E}_D[\ell(h(x), y)],$$
which searches for the hypothesis $h$ that minimizes the true risk over the data distribution.</span>

(c) In both the above settings, quantify the dependence between the risk of your algorithm and the number of training points $n$.
<span style="color:blue">Neither algorithm depends on $n$, since we know the true data distribution $D$, as can be seen in the answers above to (a) and (b).</span>

**4) Kernelizing the Perceptron [5+9=14 points]**

In this question, we will discuss the key ingredients for Kernelizing the Perceptron.

(a) Show that the Perceptron training algorithm (assuming it stops after a finite number of iterations) results in a $w$ of the form $w = \sum_{i \in [n]} \alpha_i x_i$ for some $\alpha_1, \dots, \alpha_n \in \mathbb{R}$.
<span style="color:blue">Assume each point $(x_k, y_k)$ is picked $a_k$ number of times. Since the algorithm stops we know that each $a_k$ is bounded by $T$ (the number of iterations). Then with the perceptron algorithm we can write
$$w = \sum_{i \in [n]} a_i y_i x_i; \quad b = \sum_{i \in [n]} a_i y_i$$</span>

(b) Re-write the Perceptron algorithm in a manner that the training depends on $x_i$'s only in terms of their inner products.
<span style="color:blue">Assume now instead of maintaining $w^{(t)}, b^{(t)}$, we maintain a vector $a^{(t)}$, where each index $a_i^{(t)}$ denotes the number of times perceptron picked $i$-th example.</span>

| **Algorithm 1:** Perceptron algorithm |
| :--- |
| (i); |
| **while** (ii) **do** |
| &nbsp;&nbsp;&nbsp;&nbsp;(iii); |
| &nbsp;&nbsp;&nbsp;&nbsp;$t = t + 1$; |
| **end** |
| Output $a^{(t)}$ |

<span style="color:blue">We need to change the above statement (i), (ii), and (iii) in the perceptron algorithm. Recall in the original perceptron algorithm, we have:
(i) Initialize parameters $w^{(1)} = 0, b^{(1)} = 0$ and $t = 1$
(ii) $\exists i$ such that $y_i(w^{(t)}x_i + b^{(t)}) \leq 0$;
(iii) $w^{(t+1)} = w^{(t)} + y_i x_i$; and $b^{(t+1)} = b^{(t)} + y_i$
Now we substitute the form of $w^{(t)}$ and $b^{(t)}$ from part (a) in terms of $a^{(t)}$ and maintain $a^{(t)}$ in update (iii).
(i) Initialize parameters $a_i^{(1)} = 0$ for all $i \in [n]$ and $t = 1$
(ii) $\exists i$ such that $y_i \left( \sum_{j \in [n]} a_j^{(t)} y_j k(x_i, x_j) + \sum_{j \in [n]} a_j^{(t)} y_j \right) \leq 0$
(iii) $a^{(t+1)} = a^{(t)} + e_i$, where $e_i$ is a standard basis vector for of $i^{th}$ coordinate.</span>

**5) Average margin [4+5+5+5=19 points]**

Suppose that instead of defining the margin as the distance of the nearest point to the hyperplane, you define it as the average of distances across all points. Then maximizing this notion of margin would amount to writing down the following optimization problem:
$\max_{w \in \mathbb{R}^d, b \in \mathbb{R}} \frac{1}{n} \sum_{i \in [n]} y_i(w^T x_i + b)$ subject to $\|w\|_2 = 1$. In order to make our life simpler in this exam, fix $b = 0$ so that we are only optimizing over $w$.

(a) Is this problem guaranteed to have at least one feasible solution?
<span style="color:blue">Yes, the constrain is only that the weight vector should have norm equal to 1. Such a vector exists (e.g., a standard basis). Given we have a finite number of points (with finite features), the objective will achieve finite values. Thus there is at least one feasible solution.</span>

(b) Write down the update steps of the projected stochastic gradient descent algorithm (i.e., stochastic gradient descent with a projection onto the constraint set after every iteration) for this problem.
<span style="color:blue">For a batch $B$, of size $|B|$, first, take a step in the direction of the gradient of the objective with respect to $w$ (since we are maximizing). That is, at step $t + 1$,
$$\tilde{w}_{t+1} := w_t + \frac{1}{|B|} \sum_{i \in B} y_i x_i.$$
Then to project onto the constraint set, renorm $w$ to have norm equal to 1 (divide by its current norm). That is,
$$w_{t+1} := \frac{\tilde{w}_{t+1}}{\|\tilde{w}_{t+1}\|_2}.$$</span>

(c) Does this reduce to the Perceptron algorithm or is there a difference?
<span style="color:blue">The Perceptron algorithm maximizes the distance to the *misclassified* points, not all points.</span>

(d) Please comment (qualitatively and briefly) on your perception of how well the output will perform as a classifier.
<span style="color:blue">When comparing possible hyperplanes, this objective penalizes equally a misclassified point and a point that is correctly classified but at a greater distance, since it is a (linear) signed distance function with respect to the separating hyperplane (and the correct labelling).

6) Sample complexity [5+12=17 points]

In this question you cannot use VC dimensions or the fundamental theorem of statistical learning theory or the sample complexity result on finite-sized hypothesis class that we had studied in the lecture. We would like you to take a first-principles approach by using the PAC learning formulation itself as a starting point. Consider the realizability and i.i.d. assumptions.

(a) Suppose $\mathcal{X} = \{0\}$ and $\mathcal{Y} = \{-1, 1\}$. Let $\mathcal{H}$ be all possible mappings from $\mathcal{X}$ to $\mathcal{Y}$. Consider the 0-1 loss. Find $n^*(\epsilon = 0.1, \delta = 0.1)$.
$n^* = 1$ because of the deterministic case (i.e., realizability).

(b) Answer the same question as in part (a) but now with $\mathcal{X} = \{-1, 1\}$. You can feel free to derive upper and/or lower bounds on $n^*(\epsilon = 0.1, \delta = 0.1)$. (You can restrict attention to only deterministic learning algorithms.)
Since it is a realizable case, if we observe both $x = 1$ and $x = -1$ in the training data, we will make no error. Additionally, as we have a deterministic classifier, we will assume that in the worst case we will always make an error on the unobserved sample.
Without loss of generality, we assume Bernoulli distribution on $\mathcal{X}$ with a bias $p \le 0.5$, i.e., $p(x = 1) \le 0.5$. The overall idea is the following: To make the risk greater than $\epsilon$ with probability more than or equal to $\delta$, we need to observe $n^*$ examples of a single covariate with the total probability of occurrence at least $1 - \delta$ in our training set such that expected risk on the unobserved covariate is greater than or equal to $\epsilon$.
**Case 1:** Consider $p < \epsilon$. If we observe $x = -1$ in the training set, then the risk on unseen set, i.e., $R = p$ is always less than $\epsilon$. If we observe $x = 1$ in the training set, then while the risk on the unseen set $R = 1 - p$ is greater than $\epsilon$, the probability of only observing $n^*$ samples all with $x = 1$ is $p^{n^*}$ is always less than $\delta$ for $n^* \ge 1$. Hence, in this case $n^* = 0$.
**Case 2:** Consider $p \ge \epsilon$. Then we will always have a risk greater than $\epsilon$ on the unseen point. Hence, we need to find largest $n$ such that probability of only observing all training points from a single covariate is greater than $\delta$, i.e., $p^n + (1-p)^n \ge \delta$. We will solve this by obtaining lower bound and upper bound on $n$, i.e., $2(1 - p)^n \ge \delta \ge (1 - p)^n$. This implies
$$\lfloor \log(1/\delta)/\log(1/(1 - p)) \rfloor \le n \le \lfloor \log(2/\delta)/\log(1/(1 - p)) \rfloor . \quad (1)$$
Finally, by definition of $n^*$, we have the lower bound in (1) satisfied for all $p \in [\epsilon, 0.5]$ at $n^*$. Since the lower bound on $n$ takes its maximum value at $\epsilon = p$, we get
$$\lfloor \log(1/\delta)/\log(1/(1 - \epsilon)) \rfloor \le n^* \le \lfloor \log(2/\delta)/\log(1/(1 - \epsilon)) \rfloor . \quad (2)$$
Plugging in the value of $\epsilon = 0.1$ and $\delta = 0.1$, we get $22 \le n^* \le 28$. (This last step needs a calculator and wasn't expected in the exam.)
