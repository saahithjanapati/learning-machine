# The problem

Samples are blurry, though they capture some high-level structure.

Some hypotheses for what goes wrong: 

* **Strong metric:** VAEs try to match the input distribution in KL divergence, which is quite a strong metric.
* **Poor posteriors:** The posteriors in a VAE are Gaussian – very poor modeling power, e.g. cannot model multimodal distributions.
* **Max-likelihood encourages averaging:** finding the max-likelihood $q$ to fit a distribution $p$ is equivalent to minimizing $KL(p||q)$ (by expanding the def. of $KL = E_p \log p - E_p \log q$). 

Recall from when we talked about variational methods: this KL tends to “average” modes. 

### Visual Description
The slide contains text explaining the limitations of VAEs. On the bottom right, there is a contour plot illustrating the "averaging" effect of KL divergence. A blue contour represents a multimodal distribution $p$, and a red contour represents a single-mode distribution $q$ that has been optimized to minimize $KL(p||q)$, resulting in a broad distribution that covers both modes of $p$ rather than picking one.

# DC-GAN formalization (Goodfellow et al. ‘14)

Min-max problem:
* **Min-player:** generators $g \in G$; **Max-player:** discriminators $f \in F, f(x) \in [0,1]$.
* Samples from image distr. $P_{real}$. Unif. distribution over samples: $P_{samples}$
* $P_g$ - generator distribution: $Z \sim N(0, I) \to g(Z)$

**Training loss:**
$$\min_{g \in G} \max_{f \in F} \mathbb{E}_{x \sim P_{samples}} \log f(x) + \mathbb{E}_{x \sim P_g} \log(1 - f(x))$$

* $f$ tries to output “1” for samples from $P_{samples}$, “0” for samples from $P_g$
* $f$ can be interpreted as a probabilistic classifier, penalized w/ logistic loss.

# Properties of a divergence

**Training loss:**
$$\min_{g \in G} 2 JS(P_g, p_{samples}) - \log 4$$

Jensen-Shannon divergence is a **divergence**:
$JS(p, q) \ge 0$ and $JS(p, q) = 0$ only if $p = q$

Hence, if we learn a distribution $P_g$, s.t. $JS(P_g, P_{data}) = 0$, we have $P_g = P_{data}$.

In the limit of infinite samples, $P_{real} = P_{samples}$, so if training error is 0, we have learned a distribution $P_g = P_{real}$

# What affects our choice of F?

**Statistical considerations**: very powerful discriminators (e.g. large neural networks) will require a lot of samples. Weak discriminators will specify a very weak metric: very “different” distributions will look very “similar” to metric.

**Algorithmic considerations**: if discriminators are very powerful, gradient information for generator is too weak and can vanish. If they are too weak – metric is weak.

# How to train a GAN

**“Best response dynamics”**: fix generator, find best discriminator; then fix discriminator, find best generator. Repeat.

**Better in practice**: take one gradient step for generator, do a few gradient steps for discriminator. Repeat.

Going with intuition of Wasserstein distance: we’d like the discriminators to be somewhat Lipschitz – **clipping** weights is a good idea.

# How to train a GAN

**Algorithm 1** WGAN, our proposed algorithm. All experiments in the paper used the default values $\alpha = 0.00005, c = 0.01, m = 64, n_{\text{critic}} = 5$.

**Require**: $\alpha$, the learning rate. $c$, the clipping parameter. $m$, the batch size. $n_{\text{critic}}$, the number of iterations of the critic per generator iteration.
**Require**: $w_0$, initial critic parameters. $\theta_0$, initial generator's parameters.
1: **while** $\theta$ has not converged **do**
2:   **for** $t = 0, \dots, n_{\text{critic}}$ **do** $\leftarrow$ How many discriminator gradient steps to take for each generator gradient step
3:     Sample $\{x^{(i)}\}_{i=1}^m \sim \mathbb{P}_r$ a batch from the real data.
4:     Sample $\{z^{(i)}\}_{i=1}^m \sim p(z)$ a batch of prior samples.
5:     $g_w \leftarrow \nabla_w [\frac{1}{m} \sum_{i=1}^m f_w(x^{(i)}) - \frac{1}{m} \sum_{i=1}^m f_w(g_\theta(z^{(i)}))]$ $\leftarrow$ Empirical estimates of expectations to calculate discriminator gradient $\mathbb{E}_{P_{\text{real}}}[f]$
6:     $w \leftarrow w + \alpha \cdot \text{RMSProp}(w, g_w)$
7:     $w \leftarrow \text{clip}(w, -c, c)$ $\leftarrow$ Clip
8:   **end for**
9:   Sample $\{z^{(i)}\}_{i=1}^m \sim p(z)$ a batch of prior samples.
10:  $g_\theta \leftarrow -\nabla_\theta \frac{1}{m} \sum_{i=1}^m f_w(g_\theta(z^{(i)}))$ $\leftarrow$ Generator gradient
11:  $\theta \leftarrow \theta - \alpha \cdot \text{RMSProp}(\theta, g_\theta)$
12: **end while**

Figure from Arjovsky, Chintala, Bottou ‘17

# Diagnosing small support size: bday paradox

**Birthday Paradox:**
If there are **23** people in a group, $> 1/2$ chance that two of them share a birthday.

**General version:** Suppose a distribution is uniform over $N$ images. Then $Pr[\text{sample of size } \sqrt{N} \text{ has a duplicate image}] > 1/2$.

**Birthday paradox test*** [Arora et al ICLR’18] : If a sample of size **s** has **duplicate** images with prob. $> 1/2$, then distribution essentially* has only **$s^2$ distinct images**.

**Implementation:** Draw sample of size **s**; heuristically flag possible near-duplicates. Use human in the loop to verify duplicates.

# Inception score

Suppose we use trained network – the *Inception* architecture as a **labeler** for images. Inception gives probability over labels $y$ for sample $x$: $p(y|x)$.

**Desirable features of generator:** the Inception classifier should be “sure” about the label for most images ($p(y|x)$ should have *low entropy*), and the classes it generates should be diverse ($p(y) = \mathbb{E}_{x \sim P_g} p(y|x)$ should have *high entropy*)

> Thus, we want $H(p(y|x))$ to be **low**, $H(p(y))$ is **high**.

Consider the expression: $\mathbb{E}_{x \sim P_g} KL(p(y|x) || p(y))$
$$= \mathbb{E}_{x \sim P_g} \mathbb{E}_{y \sim p(y|x)} \log p(y|x) - \log p(y)$$
$$= -\mathbb{E}_{x \sim P_g} H(p(y|x)) + H(p(y))$$

# Frechet Inception Distance (FID)

Many follow ups, e.g. *Frechet inception distance*, modified inception score, …

The Wasserstein-2 (Frechet) distance between two distributions $\mu, \nu$:
$$d_F(\mu, \nu) := \left( \inf_{\gamma \in \Gamma(\mu, \nu)} \int_{\mathbb{R}^n \times \mathbb{R}^n} \|x - y\|^2 d\gamma(x, y) \right)^{1/2},$$
where $\gamma$ is a coupling of the two distributions (= joint distribution, s.t. the marginals match $\mu, \nu$ on each coordinate).

If $\mu, \nu$ are multivariate Gaussians, this has an explicit form:
$$d_F(\mathcal{N}(\mu, \Sigma), \mathcal{N}(\mu', \Sigma'))^2 = \|\mu - \mu'\|_2^2 + \text{tr}\left( \Sigma + \Sigma' - 2(\Sigma^{1/2} \cdot \Sigma' \cdot \Sigma^{1/2})^{1/2} \right)$$

# Frechet Inception Distance (FID)

The new FID (Frechet Inception Distance) is defined as the output of the following algorithm ($f$ is a “well-trained” neural network like Inception v3):

* **INPUT** a function $f : \Omega_X \to \mathbb{R}^n$.
* **INPUT** two datasets $S, S' \subset \Omega_X$.
* Compute $f(S), f(S') \subset \mathbb{R}^n$.
* Fit two gaussian distributions $\mathcal{N}(\mu, \Sigma), \mathcal{N}(\mu', \Sigma')$, respectively for $f(S), f(S')$.
* **RETURN** $d_F(\mathcal{N}(\mu, \Sigma), \mathcal{N}(\mu', \Sigma'))^2$.

Measures “how close” the final layer distributions are (if approximated by multivariate Gaussian).
